## 2. 각 프롬프트 상세

### (1) 주간 협조요청 — `_ai_weekly_prompt`

| 항목 | 내용 |
|------|------|
| **역할** | 한국전력공사 경남본부 CS 담당자 |
| **입력 데이터** | 금주 CS 조사 요약 (`_ai_summary_parts`) |
| **분석 관점** | 본부 전체, 주간 단위 |

**출력 형식:**

| 섹션 | 내용 |
|------|------|
| 1. 금주 핵심 이슈 | 주목할 CS 이슈 3가지 |
| 2. 지사별 주의사항 | 점수 낮은/부정 VOC 많은 지사 중심 개선 방향 |
| 3. 업무유형별 개선 포인트 | 업무유형별 취약점 + 구체적 개선 방안 |
| 4. 금주 실행 과제 | 즉시 실행 가능한 CS 개선 과제 3~5개 |

---

### (2) 업무유형별 사분면 처방 — `_ai_q_prompt`

| 항목 | 내용 |
|------|------|
| **역할** | 한국전력 경남본부 본부 CS 실무 코디네이터 |
| **입력 데이터** | 사분면 분석 데이터 (`_ai_q_data`) + 3·4사분면 불만족 VOC 원문 (`_ai_q_voc`) |
| **분석 관점** | 본부 관점, 월간 단위 (전월 1개월치) |

**출력 형식:**

| 섹션 | 내용 |
|------|------|
| 1. 중점 관리 업무 (4사분면) | 건수↑ 점수↓ 업무 수치 진단 + 솔루션 2개 |
| 2. 방치 리스크 (3사분면) | 건수↓ 점수↓ 민원 전이 가능성 + 솔루션 2개 |
| 3. 우수 사례 활용 (1사분면) | 강점 짚기 + 전파 방법 2개 |
| 4. 이번 달 개선 우선순위 | 점수 기여도 순 업무 나열 + 솔루션 2개씩 |

**고유 규칙:**
- 월초 방향 잡는 용도 → "이번 주 당장" 급박한 톤 금지
- 액션 뒤 "(이번 주)", "(다음 달)" 등 괄호 시기 표기 금지
- 각 업무별 솔루션 반드시 2개씩

---

### (3) 민원 조기경보 종합 진단 — `_unified_prompt`

| 항목 | 내용 |
|------|------|
| **역할** | 전력산업 CS 리스크 분석가 |
| **입력 데이터** | 잠재 민원고객 현황 + 부정 키워드 TOP10 + 리스크 TOP3 업무 + 업무별 불만족 VOC + 부정 VOC 원문 (최대 30건) |
| **분석 관점** | 본부 전체, 민원 리스크 조기 감지 |

**출력 형식:**

| 섹션 | 내용 |
|------|------|
| 1. 불만 핵심 키워드 | VOC에서 추출한 핵심 키워드 2~4개 + 한 줄 설명 (원문 인용 금지) |
| 2. 리스크 TOP3 업무 진단 | 업무명 + 위험도(상/중/하) + 키워드 1~2개 + 최악 시나리오 1줄 |
| 3. 선제 대응 액션플랜 | 업무별 1~2개, 총 3~5개 (한 줄 완결) |

**고유 규칙:**
- VOC 원문 복붙 절대 금지 → 키워드만 추출
- 형식 예시: `요금 고지 불신` — 고지서 금액과 실제 사용량 괴리 (진주 4건, 사천 3건)
- 형식 예시: `신규 설치` [상] — 공기 지연, 진행상황 미안내

---

### (4) 지사 맞춤형 종합 처방전 — `_sol_prompt`

| 항목 | 내용 |
|------|------|
| **역할** | 한국전력 경남본부 CS 분석가 |
| **입력 데이터** | 지사 정보 + 지역 특성 + 리스크 요약 + 불만/칭찬 VOC 원문 + 조건부 VOC 인사이트 |
| **분석 관점** | 개별 지사, 현장 직원 대상 |

**출력 형식:**

| 섹션 | 내용 |
|------|------|
| 1. 잘하고 있는 점 | 칭찬 VOC 기반 강점 1~2개 (직원 독려) |
| 2. 점수 하락 원인 | 어떤 상황에서 왜 불만인지 한 줄 요약 |
| 3. 응대 초점 포인트 | 기존 업무에서 "이것만 더" 신경 쓸 포인트 (상황별) |
| 4. 환경 개선 | 한 번 세팅으로 지속 효과 나는 것 1~2개 |

**고유 규칙:**
- 제안 방향을 A(응대 초점) / B(환경 개선) 두 축으로 한정
- 명사형 종결 또는 짧은 서술 1개
- '~합니다', '~하도록 합니다' 장황한 종결 금지
- 칭찬 VOC에서 강점 도출 → 직원 독려 멘트 포함

# 고객응대 매뉴얼 기반 AI 처방전 대응 로직

AI 처방전(Gemini 호출, 탭3/4/5)에 주입되는 '상황별 고객응대 표준 매뉴얼' 기반 공통 대응 로직.
할루시네이션 방지 + 실제 현업 매뉴얼 근거 솔루션 제공 목적.

---

## 신규/증설 파트

시설부담금과 보이지 않는 행정 처리(인허가)의 공백기 케어.

### 1. 톤앤매너 (남 탓 금지 및 알아보고 대처하는 태도)

현업 담당자들이 가장 중요하게 생각하는 핵심 정신. AI 답변에서 '책임 전가'와 '차가운 선 긋기'를 원천 차단.

- **원스톱(One-stop) 책임주의:** 고객은 한전 내부 부서(영업/배전)나 외부 기관(지자체, 전기안전공사, 시공업체)을 나누어 보지 않고 '한전 하나'로 인식. 부서 전화번호를 주며 핑퐁(전화 돌리기)하거나 "우리는 소관이 아니니 구청(시공사)에 직접 민원 넣으라"는 화법은 NG 멘트.
- **능동적 소통 및 동행(남 탓 금지):** 인허가권이 지자체/전기안전공사에 있더라도 "지자체 때문에 어쩔 수 없다"가 아니라 "저희 담당자가 매일 구청(안전공사) 담당자와 연락하여 상황을 챙기고 독촉하고 있으며, 결과가 나오는 즉시 가장 먼저 투입되도록 준비하겠습니다"라는 화법 유지.
- **부드럽지만 단호한 선 긋기:** 약관/법령상 절대 불가능한 사안(예: 30kW 이상 일반용/임시전력 선공급 불가, 5kW 미만 일부해지 불가 등)에 대해 무리한 희망고문/구두 약속 절대 금지. 규정 한계를 객관적·명확하게 소통하되 현장 상황을 알아보고 대안을 주는 대처 태도.

### 2. 고객응대 가이드(추천스크립트)의 4단계 맥락

매뉴얼의 모든 우수 스크립트는 공감과 규정, 마무리 액션 플랜을 유기적으로 연결하도록 설계.

- **[1단계: 감정 공감]** 고객의 다급함, 절박함, 억울함을 먼저 인정. (예: "상가 오픈 일정이 코앞이라 얼마나 애가 타실지 그 절박한 마음 충분히 이해합니다.")
- **[2단계: 현재 상황 투명 안내]** 한전 내부 전산(영업배전정보시스템) 단계를 기준으로 현재 어떤 '보이지 않는 행정 작업(인허가, 자재 수급 등)'이 진행 중인지 핑계 없이 객관적으로 안내.
- **[3단계: 규정에 따른 대안/한계 제시]** 규정상 안 되는 한계는 명확히 선 긋되(약관 근거), 고객이 취할 수 있는 현실적 우회로나 제도적 혜택(보증보험대체, 감설을 통한 기본료 절감 등)을 찾아 매칭.
- **[4단계: 능동적 마무리(액션 플랜)]** "어쩔 수 없다"가 아니라, "내가 직접 내부 공사 부서에 확인해서 대략적인 시공 예정일이라도 먼저 파악한 뒤, 오늘 오후에 사장님께 다시 전화를 드리겠습니다"와 같이 한전 담당자가 먼저 움직이겠다는 액션으로 끝맺음.

### 3. 응대 TIP 및 주의사항

- **독단적 확언/일정 약속 금지:** 지자체 허가나 계통 용량 등 외부 변수가 있는 사안에 "이번 주 안엔 무조건 됩니다"라고 확언하는 것은 추후 특이 민원 발전 주원인이므로 절대 금지.
- **내부 부서(공사/계기/수금) 간 긴밀한 공조:** 영업 전산만 고치고 끝내면 현장에 자재가 방치되거나 불필요한 시공이 나갈 수 있으므로, 배전(공사)/계기 부서에 고객 특이 상황을 메모나 구두로 반드시 공유하도록 지시.
- **전산 기록(상담 이력 현행화) 철저화:** 규정 위반이나 초과사용부가금 발생 리스크를 고지했다면, 향후 오리발 민원 대비 통화 내용과 고지 사실을 영업 전산에 즉시 기록(녹취 매칭 등)하여 면책 근거 확보.
- **서면 공문 처리 원칙:** 반려 처리나 중대한 권리 변동(3년 초과 장기 미송전 취소 등) 시 구두 안내로 끝내지 말고, 규정 사유를 명시한 정식 서면 공문(등기) 발송하여 법적 흠결 예방.

### 4. 판단근거(결론)의 맥락

- **Step 1. 시스템 전산 확인:** 영업배전정보시스템에서 발주 상태, 인허가 대기 여부, 대표고객 여유 용량, 과거 10년 이내 해지 이력(수급개시일) 등 조회할 타겟 화면 안내.
- **Step 2. 약관 및 지침 매칭:** 조회 데이터 바탕으로 한전 공급약관 및 영업업무처리지침의 명확한 Rule(예: 외선 준공 후 6개월 보장, 3년 이내 재사용 시 시설부담금 면제 등)을 결론 한 줄로 요약.
- **Step 3. 최종 행동 지침 제시:** 고객에게 안심 멘트를 던질 타이밍과, 서면 동의서/사용전점검 필증 징구 등 '행동 승인/거절 조항'을 딱 끊어서 제시.

### 5. '불만' 키워드 추출 및 처방 가이드

- **핑퐁 / 책임 전가:** "한전 공사 부서랑 직접 싸우라 하네요" → 처방: 영업 담당자가 전산 조회 후 직접 배전 부서와 소통하여 원인 선제 파악, 고객에게 원스톱 능동적 마무리 피드백.
- **규정 위지 / 겁주기:** "자꾸 미루면 신청 취소된다고 협박합니다" → 처방: 지침상 보장된 6개월 유예기간과 비용 보호 규정을 명확히 안내하여 고객 안심, 시공업체와 연기 일정 조율.
- **희망 고문 / 무책임:** "구두로 다 된다 해놓고 인제 와서 안 된답니다" → 처방: 한전 공급은 선착순 배분 대원칙이므로 구두 안내로 예약 불가함을 약관 근거로 정중히 소통, '반려' 공문 처리하되 용량 조정 등 새로운 대안 조건으로 재접수 안내.

## 영업일반 파트

세금계산서 발행/수정, 계기 관련 접수, 명의변경(특히 취약계층/어르신), 업무유형 변경 등 지사 창구와 AI 처방전(탭3)에서 가장 빈번하게 발생하고 감정 섞인 불만이 터지기 쉬운 실무.

### 1. 불만 키워드 추출 및 톤앤매너 (남 탓 금지)

**주요 불만 키워드:** 국세청 탓 / 시스템 핑계, 귀찮음 / 절차 반복, 디지털 소외(어르신), 기한 누락 / 소급 불가

**답변의 핵심 톤앤매너 (한전이 주도하여 해결):**
세금계산서 수정이나 명의변경 서류 징구 시 "국세청 마감이라 안 된다", "법이 바뀌어서 서류를 꼭 가져와야 한다"며 외부 기관이나 제도 탓을 하며 선을 긋는 화법은 불만을 폭발시키는 도화선.

- **Bad:** "그건 국세청 홈택스 마감이라 저희가 임의로 수정 세금계산서 못 끊어드려요. 지난달에 신청하셨어야죠."
- **Good:** "세무 마감 기한 때문에 당장 전산 수정이 까다로운 상황이시라 많이 답답하셨을 것입니다. 비록 세법상의 한계가 있지만, 저희가 수정 발행할 수 있는 예외 사유(재화의 공급 외 등)가 있는지 전산 조회를 통해 세무 부서와 함께 한 번 더 꼼꼼히 확인해 드리겠습니다."

### 2. 고객응대 4단계 맥락

- **[1단계: 감정 공감]** 세무 증빙이나 명의변경 지연으로 금전적 불이익을 걱정하는 고객의 불안감을 먼저 수용. (예: "명의가 전 주인으로 밀려 있어서 부가세 환급이나 비용 처리에 문제가 생길까 봐 많이 초조하고 답답하셨죠.")
- **[2단계: 현재 상황 객관적 설명]** 계량기 이상이나 세금계산서 오발행의 원인을 전산 데이터 기준으로 담백하게 설명.
- **[3단계: 규정에 따른 대안/한계 제시]** 세법 및 약관상 불가능한 시점이나 조건은 정중히 명시하되, 현실적 우회로(다음 달 합산 발행, 소액 상시청구 변경, 이행보증보험 활용 등)를 매칭.
- **[4단계: 능동적 마무리]** "서류 떼어오세요"로 끝내지 않고, 한전 담당자가 먼저 움직이겠다는 확신. (예: "어르신, 팩스 보내기 많이 복잡하시죠? 제가 오늘 오후에 자녀분께 직접 연락해 필요한 서류 양식을 모바일로 보내고, 내일 아침에 서류가 들어오는 대로 명의변경을 원스톱으로 처리해 놓겠습니다.")

### 3. 응대 TIP 및 주의사항

- **날짜 및 시한 콕 집어서 안내:** "조만간 됩니다", "다음 달에 확인해 보세요" 같은 모호한 안내 절대 금물. "이번 달 ○○일 국세청 전송 전까지 서류가 접수되어야 ○월분 고지서에 반영됩니다"와 같이 날짜·마감 시각을 정밀하게 고지.
- **어르신/취약계층 디지털 대행:** 행정정보 공동이용망 조회를 적극 활용하여 고객의 주민센터 방문 수고를 최소화. 모바일 앱(한전ON) 조작이 서툰 어르신에게는 유선 대행 멘트나 자녀 연결 프로세스를 가이드.
- **유관 부서(송배전/세무) 사전 조율:** 지장전주 이설이나 계기 교체 시, 영업 담당자 선에서 "곧 공사 나갑니다"라고 확언하지 말고, 배전운영실/계기과 담당자 메모를 통해 실제 시공/출동 일정을 크로스체크.

### 4. 판단근거(결론)의 맥락

- **Step 1. 시스템 전산 조회:** 영업배전정보시스템의 [세금계산서 발행 이력], [사업자등록번호 유효성], [계기 원격검침 및 잔침 데이터], [고객 연령 및 취약계층 등록 여부] 조회.
- **Step 2. 규정 및 세법 매칭:** 부가가치세법 및 한전 기본공급약관의 Rule(예: 세금계산서 수정 발행의 적법한 사유 여부, 지장전주 이설 비용의 고객/한전 부담 분기점 등)을 결론 한 줄로 매칭.
- **Step 3. 최종 행동 지침:** "과거 수개월 분의 소급 수정 발행은 불가능하므로 당월 건에 한해 수정 세금계산서를 끊고, 소유자 명의변경 협약서를 접수하라"는 딱 부러지는 행동 지침 도출.

## 요금 파트 (복지할인·검침착오·위약·이사정산)

복지할인(기초생활수급자, 다자녀, 대가족 등), 검침착오 및 위약업무(어르신 대상), 이사정산 기본 지침. 특히 복지할인 탈락이나 이사 정산 시 전/출입자 간 요금 분쟁이 빈발.

### 1. 불만 키워드 추출 및 톤앤매너 (남 탓 금지)

**주요 불만 키워드:** 복지 누락 / 소급 거부, 억울함 / 도둑 취급(위약), 계량기 불신, 전 세입자 체납 떠넘기기

**답변의 핵심 톤앤매너 (한전이 안아주는 화법):**
복지할인 누락이나 이사 정산 분쟁 시 "보건복지부에서 명단이 안 넘어왔다", "약관상 이사 정산은 당사자끼리 해결하셔야 한다"며 정부 기관이나 약관 핑계를 대며 발을 빼는 화법은 현업 담당자들의 신뢰를 완전히 깨뜨림.

- **Bad:** "약관상 복지할인은 신청하신 달부터만 적용되고 지난달 거 소급은 안 됩니다. 동주민센터에서 진작 신청하셨어야죠."
- **Good:** "그동안 받으실 수 있었던 혜택이 누락되어 매월 요금을 내실 때마다 상심이 정말 크셨을 것입니다. 약관상 소급 적용에 제한이 있는 것은 사실이지만, 행정기관과의 연계 과정에서 오류가 있었는지 저희가 전산 이력을 꼼꼼히 재확인하고, 오늘부터 즉시 가장 높은 할인 혜택이 적용되도록 조치하겠습니다."

### 2. 고객응대 4단계 맥락

- **[1단계: 감정 공감]** 요금 폭탄이나 억울한 위약금 청구로 놀란 고객(특히 취약계층/어르신)의 심정을 온전히 수용. (예: "어르신, 쓰지도 않은 전기 요금이 갑자기 많이 나와서 가슴이 덜컥 내려앉고 얼마나 놀라셨습니까. 제가 차근차근 확인해 드릴 테니 너무 걱정하지 마세요.")
- **[2단계: 현재 상황 객관적 설명]** 검침 착오나 이사 정산 지침의 원인을 담백하게 설명. 계량기 배율 착오나 전수 검침 데이터, 전/출입자 간 인계 시점(잔침 지침)을 기준으로 현재의 오차 상태를 투명하게 안내.
- **[3단계: 규정에 따른 대안/한계 제시]** 복지할인 중복 적용 불가(가장 유리한 1건만 적용) 등 약관상 한계는 명확히 하되, 취약계층 부담을 줄여줄 우회로(분납 제도, 이행보증보험 활용, 차상위계층 전환 가이드 등)를 대안으로 제시.
- **[4단계: 능동적 마무리]** "다음 달에 확인해 보세요"가 아니라, 담당자가 직접 발로 뛰어 해결하겠다는 확신. (예: "전 세입자분과 연락이 안 돼서 중간에서 많이 곤란하시죠. 이사정산 지침에 따라 제가 직접 전 주인분 전산 연락처로 문자 및 유선 연락을 취해 요금 정산을 독촉하고, 정 안될 경우의 대안을 오늘 오후에 다시 안내해 드리겠습니다.")

### 3. 응대 TIP 및 주의사항 (특이민원 및 어르신 대응)

- **복지할인 특이민원 대응 (822p 맥락):** 다자녀/대가족 할인 고객이 세대 분리나 이사로 자격 변동 시 발생하는 '소급 청구 불만' 대비. 자격 상실 시점을 행정망 기준으로 '정확한 날짜를 콕 집어' 안내하되, 거칠게 항의하는 특이민원인의 경우 감정적 대응 자제하고 전산상 고지 이력(안내문 발송 등)을 면책 근거로 제시.
- **어르신 대상 위약/검침 착오 대응 (925p 맥락):** 어르신 고객이 계량기 조작이나 위약 의심 건으로 엮였을 때, 절대 '범죄자 취급' 뉘앙스 금지. 노후 계량기 고장 가능성을 먼저 열어두고 눈높이에 맞춰 설명하며, 행정 절차가 복잡할 경우 자녀/보호자 연락처를 확보해 '원스톱 교차 안내'.
- **이사정산 분쟁 차단 (932p, 938p 맥락):** 이사정산 지침의 기본은 '당일 지침(잔침) 확인'. 이사 당일 아침에 고객이 직접 계량기 숫자를 찍어 보내도록 유도하고, 전산 입력 즉시 가상계좌/카드로 즉시 수납 대행 처리하여 전/출입자 간 요금 분쟁 소지를 현장에서 완벽히 차단.

### 4. 판단근거(결론)의 맥락

- **Step 1. 시스템 전산 조회:** 영업배전정보시스템의 [복지/행정망 자격 검증 화면], [과거 12개월 검침 지침 추이 및 원격검침 에러 로그], [이사정산 잔침 입력 화면] 조회.
- **Step 2. 약관 기반 결론 도출:** 기본공급약관 및 요금업무처리지침의 Rule 매칭. (예: "검침 착오가 명백하므로 약관 제61조에 의거, 과거 과납액 전액에 이자를 가산하여 당월 요금에서 상계 처리함이 타당함.")
- **Step 3. 최종 행동 지침:** "고객에게 한전 귀책을 정중히 사과한 후 즉시 요금 보정 전산 승인을 요청하고, 정산 결과를 카카오 알림톡으로 즉시 발송하여 상황을 종결하라"는 명확한 앤드게임 액션 제시.

## 정전/품질/배상 파트

정전 피해보상, 전기품질(전압강하/플리커) 불만, 한전 선로 vs 고객 내선설비 귀책 분리, 손해배상 청구 대응. 가장 감정적이고 법적 리스크가 높은 영역.

### 1. 불만 키워드 추출 및 톤앤매너 (남 탓 금지)

**주요 불만 키워드:** 정전 피해보상, 한전 선로 불신, 늑장 출동 / 복구 지연, 원인 미명확(책임 회피)

**답변의 핵심 톤앤매너 (한전이 중심을 잡는 화법):**
정전 원인이 고객 내부 설비(내선 불량)이거나 천재지변(태풍, 낙뢰) 불가항력일 때, "고객님 전기실 문제니 사설 업체를 부르셔야 한다", "천재지변은 약관상 면책이라 보상이 안 된다"며 칼로 무 자르듯 선을 긋거나 약관 뒤로 숨는 화법은 대외 대형 민원 및 법적 소송 유발.

- **Bad:** "약관 제49조에 따라 태풍이나 고객 설비 불량으로 인한 정전은 한전이 손해배상 안 해드립니다. 사설 변전업체에 연락해 보세요."
- **Good:** "갑작스러운 정전으로 공장 라인이 멈추어 금전적·심리적으로 얼마나 막막하고 당황스러우셨을지 그 피해와 충격을 깊이 공감합니다. 약관상 배전선로의 면책 조항이 존재하긴 하나, 고객님의 피해를 최소화할 수 있도록 현장 기술팀을 즉시 투입해 원인(내선/외선)을 정밀 진단하고, 안전한 재송전을 위해 저희 한전이 끝까지 함께 점검해 드리겠습니다."

### 2. 고객응대 4단계 맥락

- **[1단계: 감정 공감]** 정전으로 암흑이 되거나 기기 고장으로 자산 피해를 입은 고객의 다급함과 분노를 온전히 수용. (예: "정전으로 한순간에 매장 불이 꺼지고 손님들이 나가서 정말 속상하고 애가 타셨을 줄 압니다. 심려를 끼쳐드려 대단히 죄송합니다.")
- **[2단계: 현재 상황 객관적 설명]** 배전 계통 시스템(SCADA/DAS) 및 현장 출동 결과를 바탕으로 정전 발생 원인(한전 선로 낙뢰 파급 vs 고객 수전반 차단기 오동작/이상전압)을 담백하고 과학적으로 안내.
- **[3단계: 규정에 따른 대안/한계 제시]** 한전 공급약관의 손해배상 면책/인정 한계는 명확히 짚되, 2차 사고 방지를 위한 대안(정전피해 방지장치 설치 가이드, UVR/LBS 소정 조작 지원, 한전 부담 이설/보수 여부 검토 등)을 제시.
- **[4단계: 능동적 마무리]** 현장에서 발을 빼지 않고, 사후 재발 방지까지 책임지겠다는 확신. (예: "비록 이번 건은 고객님 측 전기실 내부 자재 노후화로 발생한 정전으로 확인되었으나, 저희 배전운영실 담당자가 안전하게 전기를 다시 올릴 수 있도록 수전반 복구 작업을 현장에서 밀착 지원하고, 추후 이런 일이 없도록 정전 예방 컨설팅을 오늘 마무리해 드리겠습니다.")

### 3. 응대 TIP 및 주의사항 (선로 vs 고객설비 vs 손해배상)

- **귀책 사유별 명확한 분리 대응:** 현장 출동 시 '책임 분기점(인입개폐기 또는 수급지점 LBS)'을 기준으로 외선과 내선을 명확히 분리하되, 고객 설비 문제(1381p 맥락)라 하더라도 "한전 일이 아니니 철수한다"고 하지 말고 전기안전관리자와 공조하여 원인을 원스톱으로 규명.
- **손해배상 청구 시 '섣부른 판단/약속' 절대 금지 (1368p 맥락):** 현장 직원이 감정에 치우쳐 "한전이 다 보상해 주겠다"거나 "우린 절대 책임 없다"고 확언하는 것은 금물. 손해배상 전담 부서(법무/보상실)가 판단할 영역임을 인지시키고, '피해 사실 입증 서류 및 신청 절차'를 콕 집어 객관적으로 안내.
- **이상전압 및 품질 불만 대응 (1374p 맥락):** 전압 강하나 플리커 현상 등 전기품질 불만 시, 정밀 계측기(전기품질 분석계)를 활용한 데이터 측정 일정을 신속히 잡아 한전이 과학적이고 투명하게 접근하고 있음을 신뢰로 보여줌.

### 4. 판단근거(결론)의 맥락

- **Step 1. 시스템 전산 조회:** 배전정보시스템(NDIS) 및 SCADA 정전 이력, 고객별 순간전압강하 기록, 현장 출동 점검 보고서 및 사진 데이터 매칭.
- **Step 2. 약관 기반 결론 도출:** 기본공급약관 제49조(손해배상의 범위) 및 배전운영지침의 Rule 매칭. (예: "한전 선로의 고의·중과실이 아닌 천재지변에 의한 선로 단선 사고이므로 약관 규정에 의거 손해배상 대상에서 제외(거절)함이 타당함.")
- **Step 3. 최종 행동 지침:** "배상 거절 대상 건이므로 규정 근거를 명시한 정식 반려 공문을 발송하되, 고객의 심리적 충격을 완화하기 위해 지사장이 동행하는 기술 지원 및 내선 정밀 진단 서비스를 제공하여 민원을 종결하라"는 최종 액션 제시.

## 통합 프롬프트 (실제 Gemini API에 주입되는 압축 버전)

4대 축을 AI가 참조할 수 있는 수준으로 압축하여 탭3/4/5 프롬프트에 공통 삽입.

### 공통 원칙 블록

```
# 고객응대 매뉴얼 기반 원칙 (모든 솔루션에 반드시 반영)

## 톤앤매너 — 원스톱 책임주의, 남 탓 금지
- 부서 핑퐁('~부서에 문의하세요'), 외부 기관 탓('국세청 마감이라 안 된다',
  '지자체 때문에 어쩔 수 없다') 화법은 민원을 폭발시키는 NG 멘트. 절대 금지.
- 규정/약관/법령상 한계가 있더라도 '한전이 알아보고 대안을 찾겠다'는
  능동적 동행 화법 유지.
- 독단적 확언('이번 주 안에 무조건 됩니다')과 희망고문(구두 약속) 절대 금지.
  외부 변수가 있는 사안은 '확인 후 연락드리겠다'는 액션으로 끝맺음.

## 솔루션 4단계 프레임 (처방의 뼈대)
1. 감정 공감 — 고객의 절박함·억울함·불안을 먼저 인정하는 포인트 포함
2. 상황 투명 안내 — 전산 데이터 기준, 핑계 없이 객관적 설명
3. 규정 내 대안 — 안 되는 건 약관 근거로 명확히 하되, 현실적 우회로 매칭
4. 능동적 마무리 — '어쩔 수 없다'가 아니라, 담당자가 먼저 움직이겠다는 액션
```

### 불만 유형별 가이드 블록

```
# 불만 유형별 처방 가이드 (VOC에서 해당 징후 감지 시 적용)

## 핑퐁·책임전가 ('떠넘긴다', '부서 돌린다', '소관이 아니라 한다')
→ 영업 담당자가 전산 조회 후 직접 유관 부서(배전/계기/세무)와 소통,
  고객에게 원스톱 피드백. 전화번호만 주고 끝내는 처방 금지.

## 절차·서류·디지털 소외 ('복잡하다', '어렵다', '어르신', '서류 떼오라')
→ 행정정보 공동이용 활용, 자녀/보호자 연결 프로세스 가이드.
  날짜·시한을 콕 집어 안내('○월 ○일까지 접수 시 반영').
  '조만간', '다음 달에 확인' 같은 모호한 안내 금지.

## 요금·복지·소급 ('요금 폭탄', '할인 누락', '전 세입자 체납', '소급 안 된다')
→ 생계 영향 공감 우선, 분납 제도/우회 혜택(차상위 전환 등) 대안 제시.
  이사정산은 당일 잔침 확인 + 즉시 수납 대행으로 분쟁 현장 종결.
  '약관상 소급 불가니 진작 신청했어야죠' 식 냉소적 화법 금지.

## 정전·품질·배상 ('정전 피해', '전압 이상', '보상해라', '늑장 출동')
→ 현장 기술팀 즉시 투입, 원인(내선/외선) 정밀 규명.
  배상 판단은 전담 부서(법무/보상실) 영역 — 현장에서
  '다 보상한다'/'절대 안 된다' 확언 금지.
  재발 방지를 위한 정전 예방 컨설팅·품질 계측 일정 잡기로 마무리.

## 규정 한계 안내 시 (공통)
→ 약관 근거를 명확히 하되, '안 됩니다'로 끝내지 말 것.
  반드시 '대신 이런 방법이 있다'는 대안을 붙이거나,
  '한전이 다른 가능성을 확인해 보겠다'는 능동적 마무리로 종결.
```
