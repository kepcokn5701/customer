## 2. 각 프롬프트 상세

### (1) 주간 협조요청 — `_ai_weekly_prompt`

| 항목 | 내용 |
|------|------|
| **역할** | 한국전력공사 경남본부 CS 담당자 |
| **입력 데이터** | 금주 CS 조사 요약 (`_ai_summary_parts`) |
| **분석 관점** | 본부 전체, 주간 단위 |

**출력 형식:**

| 섹션 | 내용 |
|------|------|
| 1. 금주 핵심 이슈 | 주목할 CS 이슈 3가지 |
| 2. 지사별 주의사항 | 점수 낮은/부정 VOC 많은 지사 중심 개선 방향 |
| 3. 업무유형별 개선 포인트 | 업무유형별 취약점 + 구체적 개선 방안 |
| 4. 금주 실행 과제 | 즉시 실행 가능한 CS 개선 과제 3~5개 |

---

### (2) 업무유형별 사분면 처방 — `_ai_q_prompt`

| 항목 | 내용 |
|------|------|
| **역할** | 한국전력 경남본부 본부 CS 실무 코디네이터 |
| **입력 데이터** | 사분면 분석 데이터 (`_ai_q_data`) + 3·4사분면 불만족 VOC 원문 (`_ai_q_voc`) |
| **분석 관점** | 본부 관점, 월간 단위 (전월 1개월치) |

**출력 형식:**

| 섹션 | 내용 |
|------|------|
| 1. 중점 관리 업무 (4사분면) | 건수↑ 점수↓ 업무 수치 진단 + 솔루션 2개 |
| 2. 방치 리스크 (3사분면) | 건수↓ 점수↓ 민원 전이 가능성 + 솔루션 2개 |
| 3. 우수 사례 활용 (1사분면) | 강점 짚기 + 전파 방법 2개 |
| 4. 이번 달 개선 우선순위 | 점수 기여도 순 업무 나열 + 솔루션 2개씩 |

**고유 규칙:**
- 월초 방향 잡는 용도 → "이번 주 당장" 급박한 톤 금지
- 액션 뒤 "(이번 주)", "(다음 달)" 등 괄호 시기 표기 금지
- 각 업무별 솔루션 반드시 2개씩

---

### (3) 민원 조기경보 종합 진단 — `_unified_prompt`

| 항목 | 내용 |
|------|------|
| **역할** | 전력산업 CS 리스크 분석가 |
| **입력 데이터** | 잠재 민원고객 현황 + 부정 키워드 TOP10 + 리스크 TOP3 업무 + 업무별 불만족 VOC + 부정 VOC 원문 (최대 30건) |
| **분석 관점** | 본부 전체, 민원 리스크 조기 감지 |

**출력 형식:**

| 섹션 | 내용 |
|------|------|
| 1. 불만 핵심 키워드 | VOC에서 추출한 핵심 키워드 2~4개 + 한 줄 설명 (원문 인용 금지) |
| 2. 리스크 TOP3 업무 진단 | 업무명 + 위험도(상/중/하) + 키워드 1~2개 + 최악 시나리오 1줄 |
| 3. 선제 대응 액션플랜 | 업무별 1~2개, 총 3~5개 (한 줄 완결) |

**고유 규칙:**
- VOC 원문 복붙 절대 금지 → 키워드만 추출
- 형식 예시: `요금 고지 불신` — 고지서 금액과 실제 사용량 괴리 (진주 4건, 사천 3건)
- 형식 예시: `신규 설치` [상] — 공기 지연, 진행상황 미안내

---

### (4) 지사 맞춤형 종합 처방전 — `_sol_prompt`

| 항목 | 내용 |
|------|------|
| **역할** | 한국전력 경남본부 CS 분석가 |
| **입력 데이터** | 지사 정보 + 지역 특성 + 리스크 요약 + 불만/칭찬 VOC 원문 + 조건부 VOC 인사이트 |
| **분석 관점** | 개별 지사, 현장 직원 대상 |

**출력 형식:**

| 섹션 | 내용 |
|------|------|
| 1. 잘하고 있는 점 | 칭찬 VOC 기반 강점 1~2개 (직원 독려) |
| 2. 점수 하락 원인 | 어떤 상황에서 왜 불만인지 한 줄 요약 |
| 3. 응대 초점 포인트 | 기존 업무에서 "이것만 더" 신경 쓸 포인트 (상황별) |
| 4. 환경 개선 | 한 번 세팅으로 지속 효과 나는 것 1~2개 |

**고유 규칙:**
- 제안 방향을 A(응대 초점) / B(환경 개선) 두 축으로 한정
- 명사형 종결 또는 짧은 서술 1개
- '~합니다', '~하도록 합니다' 장황한 종결 금지
- 칭찬 VOC에서 강점 도출 → 직원 독려 멘트 포함

## 3. 공통 규칙 (4개 프롬프트 전부 적용)

### 현실적 제약

| 제약 사항 | 이유 |
|----------|------|
| 123 고객센터 전화 연결 지연 | 사업소 권한 밖 → 해결 불가, 제안 제외 |
| 전기요금 단가·정책·제도 변경 | 본사 권한 → 사업소에서 불가 |
| TF 구성 / 전담 창구 / 핫라인 | 소규모 사업소에서 비현실적 |
| 새 프로세스·절차·시스템 도입 | 사업소 권한 밖 |
| 문자 발송·해피콜·만족도 조사 | 이미 자동화 시행 중, 불필요 |
| 멘토링·코칭·직원 교육·롤플레이 | 직원 저격성 제안 금지 |

### 문체 규칙

| 규칙 | 설명 |
|------|------|
| 개조식(bullet) | 줄글/산문체 절대 금지 |
| 한 bullet = 한 줄 | 2줄 이상 금지 |
| VOC 원문 인용 금지 | 핵심 키워드만 추출 |
| '전기세' 금지 | 반드시 '전기요금' |
| 미사여구·AI 추임새 금지 | 인사말 없이 바로 본론 |
| 데이터 창작 금지 | 데이터에 없는 사실 금지 |

## 4. API 호출 설정

| 항목 | 값 |
|------|-----|
| API | Google Gemini (`generativelanguage.googleapis.com`) |
| 모델 우선순위 | gemini-2.5-flash → gemini-2.5-pro → gemini-2.5-flash-lite → gemma-3-12b-it |

| SSL | `ssl._create_unverified_context()` (사내 방화벽 우회) |
| 키 로딩 | 환경변수 `GEMINI_API_KEY` → `.env` 파일 fallback |
